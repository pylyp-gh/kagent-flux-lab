# kagent-flux-lab

End-to-end GitOps lab: **kind** → **Flux CD** → **Sealed Secrets** →
**agentgateway** → **kagent** → **Anthropic Claude** (з Ollama fallback).

Ціль — мінімум ручної роботи, максимум через `make` + git push. Все інфра-стан
декларативно живе в окремій GitHub репі, керується Flux.

## Топологія

```
┌─────────────────────────────────────────────────────────┐
│  kind cluster (Docker)                                  │
│  ┌──────────────────────────────────────────────────┐   │
│  │  Flux CD ◄──── git push ──── GitHub repo         │   │
│  │     │                                            │   │
│  │     ├─► sealed-secrets-controller                │   │
│  │     ├─► gateway-api CRDs                         │   │
│  │     ├─► agentgateway  ◄── LLM proxy + fallback   │   │
│  │     └─► kagent        ◄── agent framework       │   │
│  └──────────────────────────────────────────────────┘   │
│           │                                             │
│           ▼ Service type=LoadBalancer                   │
│  cloud-provider-kind (host daemon) → real LB IP         │
└─────────────────────────────────────────────────────────┘
                  │
                  ▼
        Anthropic API (primary) | Ollama localhost (fallback)
```

## Quick start

```bash
cp .envrc.example .envrc
# заповни ANTHROPIC_API_KEY, GITHUB_USER
direnv allow .

make prereqs          # перевірити tools, gh auth, env
make cluster-up       # kind + cloud-provider-kind
make flux-bootstrap   # створити GitHub репу + Flux GitOps
# далі — все через git push у Flux репу
```

## Make targets

| target                | що робить                                                |
| --------------------- | -------------------------------------------------------- |
| `make prereqs`        | перевірка tools, gh auth, env vars                       |
| `make cluster-up`     | створити kind cluster + start cloud-provider-kind daemon |
| `make flux-bootstrap` | gh repo create + flux bootstrap github                   |
| `make seal`           | kubeseal helper для секретів (stdin → SealedSecret)      |
| `make status`         | nodes + flux state                                       |
| `make cluster-down`   | прибрати все                                             |

## Структура

```
.
├── scripts/                # bash entry points (idempotent)
├── kind/cluster.yaml       # kind cluster spec
└── clusters/kind-lab/      # GitOps tree (керується Flux після bootstrap)
    ├── flux-system/        # auto-generated by flux bootstrap
    └── infrastructure/
        ├── controllers/    # HelmReleases: sealed-secrets, gateway-api, agentgateway, kagent
        ├── config/         # agentgateway LLM provider config
        └── secrets/        # SealedSecret з ANTHROPIC_API_KEY
```

## Why these choices

- **kind + cloud-provider-kind** замість minikube/k3d: Service
  `type: LoadBalancer` отримує реальний routable IP — flow як на AWS ALB, не
  псевдо-NodePort.
- **Sealed Secrets** замість External Secrets/VSO: один controller, kubeseal
  CLI, RSA-encrypted secrets можна commit'ити в Git. Мінімум infrastructure для
  GitOps secrets.
- **agentgateway** як LLM proxy: provider abstraction, fallback chains,
  observability — навіть для одної моделі дає production-grade control plane.
- **Anthropic primary + Ollama fallback**: топ tool-calling якість + offline
  safety net.
